# Проект: Прохождение частиц через изогнутую трубу

Цель проекта: разработать модель движения частиц через S-образный коридор и подобрать параметры внешнего поля так, чтобы все частицы переходили из одного конца в другой за минимальное время.

## Текущее состояние проекта
Сейчас реализован прототип второй итерации: частицы движутся внутри S-образного коридора, взаимодействуют друг с другом через потенциал Леннарда-Джонса, ускоряются внешним потенциальным полем и отображаются в окне SFML. Геометрия коридора и зон задается в условных клетках через `Config::World`, чтобы координаты было проще читать и менять.

## Глобальная цель
Дальнейшая цель проекта: развить текущую модель в кроссплатформенное приложение на C++ с SFML и добавить автоматический подбор параметров внешнего поля так, чтобы заданная доля частиц достигала целевой области за минимальное время.

## Дорожная карта

Проект развивается по итерациям. На данный момент существуют итерации:

1. `iter-1`: идеальный 2D газ в прямоугольной плоскости.
2. `iter-2-beta`: первый прототип S-образного коридора, внешнего поля, зон и камеры.
3. `iter-2`: актуальное состояние второй итерации после рефакторинга конфигурации и перехода к сеточной геометрии.

>  **Как переключиться на версию конкретной итерации:**
> `git switch --detach iter-1`, `git switch --detach iter-2-beta` или `git switch --detach iter-2`.

## Инструкция по запуску

### Предварительные требования
Для компиляции необходима библиотека **SFML 3**.
- **Linux/WSL:** установите нативную сборку SFML 3 для Linux и при необходимости передайте `-DSFML_DIR=/path/to/SFML/lib/cmake/SFML`.
- **Windows:** скачайте архив SFML 3 с [sfml-dev.org](https://www.sfml-dev.org/download/) под ваш MSVC и распакуйте, например, в `C:\Libs\SFML`.

### 🔹 Вариант Б: Windows (Visual Studio 2019/2022)

#### Шаг 1: Подготовка SFML
1. Скачайте версию SFML 3, соответствующую вашей версии Visual Studio (MSVC 16.0 для VS2019, 17.0 для VS2022) и разрядности (x64).
2. Распакуйте в путь без пробелов, например: `C:\Libs\SFML`.

#### Шаг 2: Открытие в Visual Studio
1. Запустите VS -> **"Открыть локальную папку"** (Open a local folder).
2. Выберите корень проекта (где лежит `CMakeLists.txt`).
3. VS автоматически настроит CMake.

#### Шаг 3: Настройка путей (если CMake не нашел SFML)
1. Меню **Проект** -> **Настроить параметры CMake** (Configure CMake Settings).
2. В открывшемся `CMakeSettings.json` найдите секцию `"configurations"` и добавьте строку:
   ```json
   "cmakeCommandArgs": "-DSFML_DIR=C:/Libs/SFML/lib/cmake/SFML"
   ```
   *(Замените путь на ваш реальный путь к папке SFML)*.
3. Сохраните файл.

#### Шаг 4: DLL и запуск
1. После сборки найдите `.exe` в папке `out/build/x64-Debug/`.
2. Скопируйте все `.dll` файлы из `C:\Libs\SFML\bin` в эту же папку (`out/build/x64-Debug/`).
   - Нужны DLL из SFML 3, например: `sfml-graphics-3.dll`, `sfml-window-3.dll`, `sfml-system-3.dll`.
3. Запустите `simulation.exe` через VS (правой кнопкой -> Запуск) или напрямую.

**Управление в программе:**
- Изменение размера окна: Пропорции сохраняются автоматически (черные поля).
- Перезапуск частиц: Клавиша `SPACE`.
- Масштабирование: колесо мыши, `+`, `-`.
- Перемещение камеры: зажать левую кнопку мыши и двигать мышь.
- Сброс камеры: `0`.
- Выход: Закрытие окна.

## Что уже реализовано в `iter-1`
- Генерация частиц в прямоугольной области.
- Случайная начальная скорость для каждой частицы.
- Отражение от границ с сохранением модуля скорости.
- Масштабирование окна без искажения сцены.

## Что реализовано в `iter-2`
- S-образный коридор с отражением частиц от стенок.
- Зона старта и целевая зона.
- Внешнее потенциальное поле, заданное как кусочная функция по участкам коридора.
- Вычисление силы внешнего поля как `F = -grad(U)`.
- Межчастичное взаимодействие через потенциал Леннарда-Джонса.
- Ограничение внешней силы для устойчивости численного расчета.
- Камера с масштабированием, панорамированием и сохранением пропорций окна.
- Сеточная система задания геометрии через `Config::World::gridPoint` и `Config::World::gridRect`.
- Структура `Config::Field::Params` для коэффициентов внешнего поля.
- Отдельная документация по формулам потенциалов в `docs/potentials.md`.
